# Tab reorganization to match legacy parity (B1.3) — design spec

**Phase:** Particle Editor 2026 redesign, follow-up to B1.2
**Date:** 2026-05-20
**Predecessor spec:** [2026-05-20 left-pane polish (B1.2)](2026-05-20-left-pane-polish-b1-2-design.md)
**Successor:** B1.4 (resizable splitters via `react-resizable-panels`) — deferred behind this dispatch
**Target branch:** `lt-4`
**Source proposal:** [`tasks/b1.3_legacy_parity_reorg_proposal.md`](../../../tasks/b1.3_legacy_parity_reorg_proposal.md) — 5 open questions resolved per § 4 below.

---

## 1. Why this exists

After B1.2 shipped, the user compared the new-UI Basic / Appearance / Physics
tabs against legacy Win32 screenshots and found three categories of
divergence:

1. **Section structure** — which fields live in which section, in what order.
2. **Field placement across tabs** — several fields are on the wrong tab:
   rotation fields on Basic instead of Appearance; emit-mode fields on
   Physics instead of Basic; affected-by-wind on Appearance instead of
   Physics (legacy has it on Physics, see `ParticleEditor.en.rc:350`).
3. **Generation mode** — legacy is a tri-state radio mutex (Bursts /
   Continuous / Weather) but we render it as two independent checkboxes
   plus the weather toggle sitting on Physics.

B1.3 brings the three property tabs back into per-section parity with the
legacy editor's resource layout (`IDD_EMITTER_PROPS1/2/3` in
`src/ParticleEditor.en.rc`), **without** changing the underlying schema or
bridge surface. Schema already carries every field needed; only the UI
mapping changes.

A bundled correctness fix piggybacks on the dispatch: the percent-display
inversion legacy applies to `randomLifetimePerc` and `randomScalePerc`
(legacy `Emitter.cpp:487, 492`) isn't applied in the new UI today, so the
current form interprets a `0..1` engine float as if it were already a
`0..100` percent. Discovered during B1.3 prep; fix-bundling avoids
re-churning the same fields in a follow-up.

---

## 2. Goal

Single sentence: **the three property tabs render with legacy section
structure, legacy field placement, a tri-state Generation radio mutex, and
trailing-colon labels — using the B1.2 collapsible `Section` primitive as
the section chrome.**

## 3. Scope

### 3.1 In scope (B1.3)

1. **Tri-state Generation radio mutex.** Three radios (Bursts / Continuous
   stream / Weather particle) deriving from `(useBursts,
   isWeatherParticle)`. Atomic two-field bridge patches on click. All
   three branches' sub-fields always rendered, `disabled` on inactive
   branches.

2. **Basic tab restructure** to match `IDD_EMITTER_PROPS1`
   (`src/ParticleEditor.en.rc:426-477`):
   - Sections: Emitter Timing / Generation / Connection.
   - Generation includes Weather sub-block (cube size, distance from
     camera, "Particles" spinner that mirrors `nParticlesPerSecond`).
   - Generation includes Maximum lifetime + Minimum lifetime % (moved
     here from Emitter Timing in the new UI; matches legacy).
   - Connection includes Emit mode + Emit offset (moved from Physics).
   - Drop from Basic: rotation fields (4), parent link strength, random
     scale, Index.

3. **Appearance tab restructure** to match `IDD_EMITTER_PROPS2`
   (`src/ParticleEditor.en.rc:376-424`):
   - Sections: Textures / Random color addition / Tail / Rotation /
     Rendering.
   - Textures: Color texture, Bump texture (label rename: Normal →
     Bump), Texture elements (`textureSize`), Minimum scale %
     (`randomScalePerc`, **inverted display**).
   - Drop the `nTriangles` and `Affected by Wind` rows from Appearance.
   - Move rotation fields here (Random rotation direction, Fixed
     random rotation, ° and ± ° sub-spinners).
   - Rendering: "Always face camera" (renamed from World Oriented;
     semantic flip — legacy checks `!isWorldOriented`), Heat particle,
     No depth test, Blend mode.

4. **Physics tab restructure** to match `IDD_EMITTER_PROPS3`
   (`src/ParticleEditor.en.rc:338-374`):
   - Sections: Initial position (`groups[2]`) / Initial speed
     (`groups[0]`) / Acceleration / Ground interaction.
   - Initial speed section also hosts Inward speed, Parent speed
     inherit (moved from Basic), Affected by wind (moved from
     Appearance).
   - Acceleration section: X/Y/Z, Gravity acceleration, Inward
     acceleration, Object space acceleration.
   - Ground interaction: Behavior + Bounciness.
   - Drop from Physics: Emit mode (moved to Basic), Weather toggle and
     weather sub-fields (moved to Basic > Generation), `groups[1]`
     (Lifetime random-param) entirely.

5. **`displayInvertedPercent` prop on `FieldSpinner`** that handles the
   `value` → `100 - value*100` display transform and the inverse
   `(100 - displayed) / 100` commit transform. Applied to two fields:
   - `randomLifetimePerc` (legacy "Minimum lifetime:" with "%" unit)
   - `randomScalePerc` (legacy "Minimum scale:" with "%" unit)

6. **Trailing colons on field labels.** Match legacy convention
   ("Initial spawn delay:", "Cube size:", "Bounciness:" etc.). Section
   titles stay colon-less (legacy GROUPBOX titles don't use colons —
   verified against `ParticleEditor.en.rc:448-450`).

7. **`groups[]` chrome unification.** Drop the `<fieldset>` + `<legend>`
   bordered shell at `EmitterPropertyTabs.tsx:1046-1053`; render
   `Initial position` / `Initial speed` inside the same collapsible
   `Section` primitive as the other inspector sections.

8. **Test corpus updates.** Estimated 25–40 specs need label-text edits
   (mostly `getByLabelText("Old Label")` → `getByLabelText("New
   Label:")`) plus tab-membership relocations for the 5 moved fields.
   New specs for the tri-state Generation mutex and the
   `displayInvertedPercent` round-trip.

9. **Docs.** `CHANGELOG.md` (B1.3 entry, conventions per `CLAUDE.md`),
   `tasks/HANDOFF.md` (next-session pointer to B1.4), `ROADMAP.md`
   (strikethrough + ✅ Shipped under [TIER-K] tag for the parity-reorg
   item; B1.4 stays unshipped).

### 3.2 Out of scope (deferred)

- **Bridge schema changes.** None anticipated. All field moves and
  drops are UI-only; serializer / dispatcher remain untouched.
- **C++ work.** None — every schema field is already on the wire from
  `BridgeDispatcher.cpp:1998-2050`. Verified via grep.
- **Engine behaviour changes.** Dropped fields (`nTriangles`,
  `weatherFadeoutDistance`, `groups[1]`, `index`) stay in the schema
  and on the binary serializer (ParticleSystem chunks 0x05, 0x40,
  groups array, etc.). Loading an existing `.alo` file with non-default
  values for these fields preserves them across save round-trips even
  though there's no edit affordance.
- **Resizable splitters (B1.4).** Deferred. Renames the right panel
  border drag UX; orthogonal to tab content.
- **Appearance + Physics live wiring (B2).** Already wired (read & write)
  per `EmitterPropertyTabs.tsx:855-1022`. The "B2 wiring" follow-up was
  filed when those tabs were placeholder; it's now obsolete except for
  any new fields introduced by B1.3 (none — every B1.3 field is already
  bound on one of the three tabs today).

### 3.3 Out-of-scope rationale

- **Re-introducing a GROUPBOX bordered outline** around sections: B1.2
  just shipped collapsible chevron `Section` primitive across the
  inspector. Adding borders would regress B1.2's visual decisions.
  Section titles + hairline divider carry the role of "grouping"
  visually; users coming from legacy will still recognize structure.
- **Restoring per-row `index` editor as a small "Atlas" section** (Q1
  option B): rejected per user. Drop entirely. Schema retains; the
  field exits the inspector with no replacement.

## 5. Architectural approach

### 5.1 Tri-state Generation mutex

State derived in `BasicTab`:

```ts
type GenerationMode = "bursts" | "continuous" | "weather";

const mode: GenerationMode =
  properties.isWeatherParticle ? "weather"
  : properties.useBursts        ? "bursts"
  :                               "continuous";

const setMode = (next: GenerationMode) => {
  switch (next) {
    case "bursts":     onCommit({ useBursts: true,  isWeatherParticle: false }); break;
    case "continuous": onCommit({ useBursts: false, isWeatherParticle: false }); break;
    case "weather":    onCommit({ isWeatherParticle: true }); break;
  }
};
```

Each mode emits **one** bridge patch (the dispatcher merges the two-key
patch atomically); the engine sees a consistent `(useBursts,
isWeatherParticle)` pair after each click.

Three new radio rows under the Generation section header. Below each
radio: its sub-fields, always rendered, `disabled={mode !== "bursts"}`
(etc.). The existing `Spinner` + `FieldSpinner` `disabled` plumbing
already handles the visual cascade. The Maximum / Minimum lifetime
fields render below all three radios as always-enabled fields.

**Radio primitive.** Use Radix `<RadioGroup>` (already on `@radix-ui`)
or hand-roll three `<button role="radio">` rows. Hand-rolled keeps the
visual fidelity tight to the legacy three-row stack and avoids fighting
Radix's vertical grouping defaults. Decision: hand-rolled, styled with
`.form-row` for label alignment + a small radio dot circle to the left
of each.

### 5.2 `displayInvertedPercent` on `FieldSpinner`

New prop on the existing `FieldSpinner` primitive at
`EmitterPropertyTabs.tsx:532-572`. When set:
- **Display**: `value={Math.round(100 - props.value * 100)}` (the
  Spinner's internal state).
- **Commit**: `onCommit((100 - displayed) / 100)`.
- `min={0}` and `max={100}` enforced internally; caller doesn't pass
  them.

Math is bounded to 0–100 by Spinner's `min`/`max` clamping. The two
adopting fields:

```tsx
<FieldSpinner
  label="Minimum lifetime:"
  value={properties.randomLifetimePerc}
  displayInvertedPercent
  unit="%"
  onCommit={(v) => onCommit({ randomLifetimePerc: v })}
/>
<FieldSpinner
  label="Minimum scale:"
  value={properties.randomScalePerc}
  displayInvertedPercent
  unit="%"
  onCommit={(v) => onCommit({ randomScalePerc: v })}
/>
```

Specs: a small unit suite covering value-display ↔ commit round-trip
math at the boundary values (0, 50, 100) and at non-integer engine
values (verify the `Math.round` matches legacy behaviour).

### 5.3 `groups[]` section integration

`groups[0]` and `groups[2]` move into the section structure as full
`<Section title="Initial position:">` / `<Section title="Initial
speed:">` wrappers. The internal `GroupSection` component renaming to
`GroupBody` (it no longer owns the `fieldset`/`legend` chrome — the
parent `Section` does):

```tsx
<Section title="Initial position:">
  <GroupBody group={properties.groups[2]} onChange={(p) => updateGroup(2, p)} groupId={2} />
</Section>

<Section title="Initial speed:">
  <GroupBody group={properties.groups[0]} onChange={(p) => updateGroup(0, p)} groupId={0} />
  <FieldSpinner label="Inward speed:" ... />
  <FieldSpinner label="Parent speed inherit:" ... />
  <FieldCheckbox label="Affected by wind" ... />
</Section>
```

`GroupBody` keeps its per-type conditional rendering (`type === GT_BOX`
shows Min/Max Vec3Rows, etc.). The aria-label prefixes ("Group 1
Value X", "Group 3 Min Z") stay stable so existing specs continue to
match by selector.

`groups[1]` (Lifetime) is dropped from the render tree entirely. The
schema-side `.groups[1]` data passes through unchanged on read/write
round-trips since the schema still carries 3 entries.

## 6. Risks named up front + mitigations

1. **Two-key atomic patch reaches the dispatcher as two sequential
   writes.** `commit()` in `EmitterPropertyTabs.tsx:196-214` applies
   the optimistic local update and sends one bridge `request()`. As
   long as we pass both keys in a single object, both `if
   (patch.contains("useBursts"))` and `if
   (patch.contains("isWeatherParticle"))` branches at
   `BridgeDispatcher.cpp:2110-2117` fire on the same patch object,
   resulting in one frame of engine state change. **Mitigation**:
   verify the patch shape in a new vitest spec — assert the mock
   bridge receives one `set-properties` call with both keys, not two
   sequential calls.

2. **`Always face camera` semantic flip vs `isWorldOriented`.** Legacy
   checks `!isWorldOriented`; new label inverts the meaning. The
   schema name `isWorldOriented` stays. **Mitigation**: in `BasicTab`
   or `AppearanceTab`, the checkbox `checked={!properties.isWorldOriented}`
   and `onCheckedChange={(v) => onCommit({ isWorldOriented: !v })}`.
   Inline inversion; no new schema work. A new spec covers the flip
   round-trip.

3. **`BLEND_BUMP` force-face cascade interaction with the new
   "Always face camera" label.** The existing logic at
   `EmitterPropertyTabs.tsx:689` (`forceFace = blendMode === BLEND_BUMP`)
   currently forces `World Oriented` to display as unchecked. With the
   label inversion, this needs to flip to: when `forceFace` is true,
   display "Always face camera" as **checked + disabled** (camera is
   forced to face). **Mitigation**: rewrite the cascade in the
   AppearanceTab > Rendering section with the inverse logic explicit.
   Spec: 2 cases — `blendMode = ADDITIVE` shows the checkbox
   responsive to `isWorldOriented`; `blendMode = BLEND_BUMP` shows it
   forced-checked + disabled.

4. **Weather mode "Particles" spinner aria-label collision.** Both
   the Continuous-mode spinner and the Weather-mode spinner bind to
   `properties.nParticlesPerSecond`. They render simultaneously
   (all-rendered-disabled policy). If both have `aria-label="Particles
   / second:"`, `getByLabelText` returns ambiguous. **Mitigation**: use
   distinct aria-labels — `"Particles/second:"` for the Continuous
   spinner and `"Particles:"` for the Weather spinner. The schema
   field is the same; only the label differs. Specs assert by which
   `within(radioBranch)` they live in to avoid the duplicate-label
   issue.

5. **`groups[1]` data preservation across drop.** The schema still
   carries `properties.groups[1]` on read; we just don't render it. If
   the user opens an `.alo` file with non-default `groups[1]` values,
   they pass through untouched on save. **Mitigation**: a vitest spec
   that loads a fixture with `groups[1] = { type: GT_BOX, min: ... }`,
   then mounts the PhysicsTab, then triggers a no-op commit (or no
   commit at all), and asserts the round-tripped DTO still has the
   `groups[1]` block intact.

6. **Section-default-open state churn.** B1.2 shipped `Section` with
   `defaultOpen={true}` and per-mount state. When the user collapses
   a section, switches emitters, the section re-mounts open. This is
   a B1.2 bug that B1.3 inherits — accept as-is; out of scope.
   **Mitigation**: nothing. Document in HANDOFF as a known follow-up.

7. **Inverted-percent commit precision drift.** `Math.round` on the
   display side + float division on the commit side can introduce
   1-percent rounding error after many round-trips (display→commit→
   display). Legacy has the same property — `(100 - perc*100)` as int
   on display, `(100 - GetUIInteger) / 100.0f` on commit, no
   precision-preservation pass. **Mitigation**: not worth designing
   around; matches legacy exactly. Document in the
   `displayInvertedPercent` JSDoc comment.

## 7. Manual verification checklist

### Happy paths

- [ ] Open the editor; select an emitter; observe Basic tab structure:
      Name row at top, three Sections (Emitter Timing, Generation,
      Connection), Generation contains tri-state radios with sub-fields.
- [ ] Click each Generation radio in turn; verify the patch is atomic
      (two-key for Bursts/Continuous; one-key for Weather);
      sub-fields in inactive branches show as `disabled`.
- [ ] Switch to Appearance tab; verify five sections: Textures, Random
      color addition, Tail, Rotation, Rendering. No `nTriangles` or
      `Affected by Wind` rows.
- [ ] Switch to Physics tab; verify four sections: Initial position,
      Initial speed, Acceleration, Ground interaction. No emit mode,
      no weather fields, no third-group section.
- [ ] Toggle `Affected by wind` on Physics → Initial speed; verify
      the value round-trips to engine and back.
- [ ] Toggle `Always face camera` (Appearance > Rendering); verify
      semantic flip lands at the schema level (`isWorldOriented`
      stores the *negation* of the checkbox state).
- [ ] Edit Minimum lifetime % → 75%; verify the engine receives
      `randomLifetimePerc = 0.25` (100-75 = 25, /100 = 0.25).
- [ ] Edit Minimum scale % → 75%; verify the engine receives
      `randomScalePerc = 0.25`.
- [ ] Set blend mode to Bump map; verify "Always face camera"
      becomes checked + disabled.

### Edge cases

- [ ] Open an `.alo` with `groups[1]` non-default; verify
      `PhysicsTab` renders nothing for it but the value is preserved
      on save.
- [ ] Open an `.alo` with `nTriangles = 5`; verify Appearance tab
      doesn't show a Triangles row but the value is preserved on save.
- [ ] Open an `.alo` with `index = 7`; verify Basic tab doesn't show
      an Index row but the value is preserved on save.
- [ ] Click Weather radio while in Bursts mode; verify the engine
      sees `isWeatherParticle = true` but `useBursts` remains `true`
      (preserved). Switch back to Bursts; verify burst sub-fields
      re-enable.
- [ ] Select two emitters; switch tabs; switch emitters; verify the
      newly-selected emitter's tab structure is identical and the
      tri-state radio reflects the *new* emitter's mode.

### Refused inputs / boundaries

- [ ] Enter `0` in Minimum lifetime %; verify commit is
      `randomLifetimePerc = 1.0`. Enter `100`; verify
      `randomLifetimePerc = 0.0`.
- [ ] Spinner clamp: enter `-5` in any percent field; verify clamp to
      `0`. Enter `150`; verify clamp to `100`.

### Cleanup / regression

- [ ] No console errors during any of the above.
- [ ] Legacy `--legacy-ui` Win32 mode unaffected — open the editor in
      legacy mode and verify Basic/Appearance/Physics dialogs are
      unchanged.
- [ ] Vitest 254 → expected ~265–275 (range — depends on exact
      additions). All passing.
- [ ] Playwright 83/83 passing (no spec count change expected; the
      native spec suite is at a structural level).
- [ ] MSBuild Debug x64 clean.
- [ ] `pnpm build` clean (no TypeScript errors from the
      `FieldSpinner` prop signature change).

### Debug instrumentation

None planned. The diff is pure CSS/JSX/test reshuffle; no new event
